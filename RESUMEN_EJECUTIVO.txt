================================================================================

PROYECTO: BioSim - Simulador Algorítmico de Epidemias
LENGUAJE: C99
POBLACIÓN INICIAL: 1000 individuos (10 infectados)
DURACIÓN SIMULACIÓN: 60 días

================================================================================
ESTRUCTURA DE LOS 8 SUBPROBLEMAS
================================================================================

┌─ FASE 1: ANÁLISIS INICIAL (Población inicial)
│
├─ [SP1] ANALISIS DE DATOS ─────────────────────────────────────────────────
│  Qué hace: Ordena individuos por 3 criterios diferentes
│  Algoritmos: MergeSort O(n log n)
│              QuickSort O(n log n) promedio
│              HeapSort O(n log n)
│  Usa: Poblacion = 1000 individuos (10 infectados)
│  Resultado: Ranking de riesgos
│  
├─ [SP2] DETECCION DE BROTES ────────────────────────────────────────────────
│  Qué hace: Detecta dónde empiezan los focos epidemiológicos
│  Algoritmos: BFS O(V+E)
│              DFS O(V+E)
│  Usa: Poblacion = 1000 individuos (10 infectados iniciales)
│  Resultado: 8 componentes conectadas
│             Cuáles territorios están en riesgo inmediato
│
├─ [SP3] PROPAGACION TEMPORAL ───────────────────────────────────────────────
│  Qué hace: Simula qué hubiera pasado en 60 días
│  Algoritmo: Min-Heap para procesar eventos O(n log n)
│  Usa: Poblacion = 1000 individuos (10 infectados iniciales)
│  Resultado: Estadísticas de propagación simulada
│             ~603 infectados hipotéticos después de 60 días
│             NOTA: NO modifica poblacion[], solo simula
│
│                    ↓ ACTUALIZACIÓN DE POBLACIÓN
│                    Simula el escenario realista
│                    ~15% de sanos se exponen → infectan
│                    Nuevos infectados: 166 (aproximadamente)
│
└─ FASE 2: ANÁLISIS CON ESCENARIO POST-PROPAGACIÓN
  
├─ [SP4] MINIMIZACION DE RIESGO ────────────────────────────────────────────
│  Qué hace: Decide a quién aislar para minimizar contagios
│  Algoritmo: Greedy con ordenamiento O(n log n)
│  Usa: Poblacion ACTUALIZADA (166 infectados)
│  Resultado: Eficacia de aislar 5%, 10%, 20%
│             - 5% (50 personas): Reduce 37% del riesgo
│             - 10% (100 personas): Reduce 68% del riesgo
│             - 20% (200 personas): Reduce 100% del riesgo
│
├─ [SP5] RUTAS CRITICAS ─────────────────────────────────────────────────────
│  Qué hace: Encuentra caminos de máxima probabilidad de contagio
│  Algoritmo: Dijkstra modificado con Max-Heap O((n+m) log n)
│  Usa: Grafo de territorios + Cepas (NO usa población)
│  Resultado: Rutas más peligrosas entre territorios
│             Ejemplo: T3 → T9 → T13 → T16 → T15 (prob. 2.67%)
│
├─ [SP6] CONTENCION (VACUNACION) ────────────────────────────────────────────
│  Qué hace: Planifica red mínima de cobertura de vacunación
│  Algoritmos: Kruskal (Union-Find) O(m log m)
│              Prim (con Heap) O((n+m) log n)
│  Usa: Grafo de territorios (NO usa población)
│  Resultado: MST (Árbol de Expansión Mínima)
│             Define territorios críticos para vacunación
│
├─ [SP7] CLUSTERING DE CEPAS ────────────────────────────────────────────────
│  Qué hace: Agrupa variantes por similitud de ADN
│  Algoritmo: Trie para búsqueda de prefijos O(k*L)
│  Usa: Array de 50 cepas (NO usa población)
│  Resultado: Familias de variantes del virus
│             Agrupa por patrones de ADN comunes
│
└─ [SP8] CONSULTAS RAPIDAS ─────────────────────────────────────────────────
   Qué hace: Búsquedas rápidas de información de pacientes
   Algoritmo: Tabla Hash O(1) promedio
   Usa: Poblacion ACTUALIZADA (166 infectados)
   Resultado: Historial rápido de cambios de estado
              Consultas por ID de individuo
              Información de territorios

================================================================================
COMPLEJIDADES ALGORÍTMICAS
================================================================================

[SP1] ANALISIS DE DATOS
  MergeSort:  O(n log n)     Garantizado
  QuickSort:  O(n log n)     Promedio, O(n²) peor
  HeapSort:   O(n log n)     Garantizado
  Total: O(3 * n log n) = O(n log n)

[SP2] DETECCION DE BROTES
  BFS:        O(V + E)       V=infectados, E=contactos
  DFS:        O(V + E)       V=infectados, E=contactos
  Total: O(2 * (V + E)) = O(V + E)

[SP3] PROPAGACION TEMPORAL
  Generar eventos:   O(n)
  Procesar eventos:  O(n log n)  Con Min-Heap
  Total: O(n log n)  donde n = número de eventos

[SP4] MINIMIZACION DE RIESGO
  Calcular riesgos:  O(n)
  Ordenar:          O(n log n)
  Seleccionar:      O(k)
  Total: O(n log n)

[SP5] RUTAS CRITICAS
  Dijkstra:         O((n + m) log n)  Con Max-Heap
  
[SP6] CONTENCION
  Kruskal:          O(m log m)        Con Union-Find
  Prim:             O((n + m) log n)  Con Heap

[SP7] CLUSTERING
  Construir Trie:   O(k * L)   k=cepas, L=longitud ADN
  Búsquedas:        O(L + m)   m=resultados

[SP8] CONSULTAS
  Construir Hash:   O(n)       n=individuos
  Búsquedas:        O(1)       Promedio
  Historial:        O(k)       k=cambios
